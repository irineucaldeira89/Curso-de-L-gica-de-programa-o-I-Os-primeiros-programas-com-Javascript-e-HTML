Vejamos o exemplo do programa entendendo_dois_mundos.html. Seu conteúdo é: Observe a instrução document.write que recebe entre () o texto: "Estou escrevendo do mundo JavaScript no mundo HTML". Quando usamos document.write, dentro de script, estamos, na verdade, escrevendo no mundo HTML, ou seja, o que veremos é um resultado HTML. Para quem está começando pode parecer estranho, visto que conseguimos fazer a mesma coisa escrevendo o texto diretamente no mundo HTML como no exemplo abaixo: Estou escrevendo do mundo JavaScript no mundo HTML Lembre-se que tudo que estiver entre as TAG's é considerado pelo navegador como código JavaScript e tudo que estiver fora das TAG's, não importa em que lugar, é código HTML. Mais um exemplo: Estou escrevendo do mundo JavaScript no mundo HTML Aqui também é mundo HTML, porque esta fora da TAG script A vantagem é que com document.write podemos passar o resultado de um cálculo entre outras coisas de maneira dinâmica já que o mundo HTML não é capaz de realizar operações matemáticas. Desse modo, podemos fazer o seguinte: Nesse caso, o document.write recebe dentro de () o resultado da operação que é 30. O que o document.write faz é alterar a página de maneira a escrever o que quiser no mundo HTML. Isso que estamos escrevendo pode ser o resultado de uma lógica de programação mais complexa! Quer mais uma prova de que document.write pertence ao mundo JavaScript e consegue escrever no mundo HTML? Vejamos mais um exemplo: Veja que estamos escrevendo do mundo JavaScript no mundo HTML o texto "

Seja bem-vindo

". Se isso faz parte do mundo HTML, o navegador irá interpretar o código como HTML e colocará o texto em destaque pois usamos a TAG

. O resultado é o mesmo que se tivéssemos escrito diretamente no mundo HTML:

Seja bem-vindo

Por fim, não podemos escrever no mundo HTML instruções JavaScript: document.write("Seja bem-vindo"); O HTML não entende que deve exibir o texto passado pela instrução, ele imprime a instrução e não seu resultado!
